# LR1 — Программа для шифрования на основе (3)DES

Кратко

Это учебная реализация алгоритма 3DES (Triple DES) в виде CLI-приложения на Go.
Проект предоставляет команды `encode` и `decode` для шифрования и расшифровки строк.

Особенности

- Реализованы блоковые операции DES (IP, FP, S-боксы, P, расширение, генерация ключей и т.д.).
- Поддерживается схема 3DES: E_k3(D_k2(E_k1(plaintext))).
- Вывод шифротекста в Base64.
- Возможность передать собственные 64-битные ключи (key1, key2, key3) или воспользоваться автоматически сгенерированными.
- Флаг `info` для печати статистики (корреляция и распределение битов).

Сборка

Сборка бинарника в корне репозитория:

```bash
go build -o lr1
```

Запуск без сборки (go run):

```bash
go run . encode "Hello, world!"
```

Команды

1) encode

Шифрует переданную строку и выводит результат в Base64.

Синтаксис:

```bash
./lr1 encode <текст> [--key1=<uint64>] [--key2=<uint64>] [--key3=<uint64>] [--sk] [--info]
```

Флаги:
- --key1, --key2, --key3 — 64-битные ключи шифрования (uint64). Если не заданы, генерируются случайные.
- --sk — вывести в консоль сгенерированные ключи.
- --info — вывести статистику (коэффициент корреляции между входными и выходными данными и распределение битов).

Примеры:

1) Шифрование с автоматически сгенерированными ключами:

```bash
./lr1 encode "Secret message"
# Вывод будет вида:
# encode: <BASE64-строка>
```

2) Шифрование с указанными ключами и выводом статистики:

```bash
./lr1 encode "Secret message" --key1=12345 --key2=67890 --key3=13579 --sk --info
```

2) decode

Декодирует Base64-строку, полученную в результате работы `encode`, и выводит исходный текст.

Синтаксис:

```bash
./lr1 decode <base64-строка> [--key1=<uint64>] [--key2=<uint64>] [--key3=<uint64>] [--sk]
```

Флаги те же, что и для `encode`. Важно: для успешной расшифровки нужно передать те же ключи, что использовались при шифровании.

Примеры:

```bash
# Предположим, encode вывел: encode: AbCdEfGh...
./lr1 decode "AbCdEfGh..." --key1=12345 --key2=67890 --key3=13579
# Вывод: encode: Secret message
```

Примечания

- Обратите внимание: текущая реализация в выводе использует префикс `encode:` и для команды `decode` (это строка, напечатанная программой), поэтому результат шага декодирования также будет помечен как `encode:`.
- Ключи — 64-битные числа. В реальных приложениях для безопасности рекомендуется использовать проверенные криптографические библиотеки вместо учебной реализации.
